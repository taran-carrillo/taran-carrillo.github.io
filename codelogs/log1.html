Code Log 1: HTML Semantic Tags

What is the difference between non-semantic and semantic tags in HTML?

The difference is that semantic tags clearly define something specific whereas the non-semantic tags don't clearly define anything.

What are two (or more) benefits of using semantic tags?

  1. Legibility would be the first benefit, it helps to keep the work space clear and organized. It also provides a clear definition of what the purpose of each line is. Whether it is a header or a body paragraph, a general main portion or a specific section.
  2. The second benefit would be that it makes the code more machine-readable, it gives the computer a clear outline of how the website should be structured.
