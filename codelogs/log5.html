Code Log 5: Images

What are the differences between presenting an image using an HTML tag (like ) and presenting it as the background of an HTML element (using the CSS property background-image)? Think in terms of differences in code as well as differences in functionality and layout.

  • When using the <img> tag, the image isn't inserted into a web page, it is linked to it using the reference source "src" that brings the image URL over to the web page. The background image applies a gradient or specific image over the existing background in the webpage that can be controlled in width and height to define how much of the page it covers. The background-image tag uses a url() tag to link an image to the webpage.

Besides background-image, what other CSS properties affect the display of background images? Describe at least two more CSS properties.

  • The alt="" tag defines a text alternative to the linked image for if the image doesn't load or if the user is using a text reader for the webpage.
  • You can also define the width and height in the same <img> tag, you can do this using individual width= and height= sections, or you can do a style="" that can include the width and height in it. The difference between the two is that when using the style="" tag it prevents the stylesheet from affecting the shape of the image.

In one sentence, how would you define the differences between dimensions and resolution in a bitmap image?

  • Resolution should be based on the dimension of the desired image size, dimensions are the width and height of an image, it tells the bitmap how to lay the image, resolution is how many pixels the image is going to use when appearing on a screen.
